## チャッピー回答メモ（受信側: PC→WOBC）

受信側（PC→WOBC）は、基本的に「WCPPパケットをシリアルから取り込み、検証してカーネルへ流す」機能。

### 実装されている機能

- 受信本体は `src/library/core/serial_bus.cpp` の `SerialBus::loop()` にある。
- 1バイトずつ受信し、終端 `0x00` が来るまでバッファリングする（途中の `0x00` はサイズ不一致なら継続読み取り）。
- フレーム検証を実施する。
	- 長すぎる受信: `sbRL`
	- サイズ不一致: `sbSM`
	- CRC不一致: `sbCS`
- 正常時は `decodePacket()` でパケット化し、`sendPacket()` でカーネル配信する。
- 送信側への即時ループバックを避けるため、`all_packets` を `exclude` 指定して再配信する。

### PCから使うときの実用フォーマット

- 送る単位は 1フレーム = パケット本体 + CRC8 + 終端 `0x00`。
- CRC8 は WCPP の `checksum` と同じ（`wcpp::Packet::checksum`）。
- Python実装でも同じ送信をしている（`src/library/wcpp/python/util.py`）。
- 受信バッファ上限は `wcpp::size_max + 2`（255+2）。

### 何ができるか（使い方）

- PCからコマンドパケットを送って、各コンポーネントの command listener に届けられる。
- `Component` は command + `component_id` で受信待ちする実装。
- ローカル形式（`origin=0x00`）でも通せるため、まずはローカルコマンドで機能試験がしやすい。
- テレメトリ形式パケットも注入できるため、GS/可視化側の動作試験にも使える。
